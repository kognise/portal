Project Portal: Ping

A service to keep websites hosted on Now.sh, Repl.it, Glitch.me, and similar awake. Why should you use this instead of Uptime Robot? Because Ping is built for the purpose of keeping sites awake, not for monitoring uptime. We guarentee reliability: your website will always be pinged on time.

Endpoints

If you want, we have an API as well. These are it's endpoints.

  • GET /ping: shows this very page
  • GET /ping/:site: enables pinging for a site; responds with status 409 if the site is already set to ping
  • GET /ping/:site/stop: stops pinging a site
  • GET /ping/:site/last: responds with the last time a site was pinged
